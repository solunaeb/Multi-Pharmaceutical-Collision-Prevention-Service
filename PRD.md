# PRD: 다제약물 충돌 방지 서비스

## Problem — What are you solving?

한국에서 10종 이상 약물을 60일 이상 복용하는 만성질환자가 171만 명(2025년 기준)에 달하지만, 환자 스스로 약물 간 충돌을 확인할 수단이 없다.

핵심 원인은 **세 겹의 약물 사각지대**다:

1. **처방약 ↔ 처방약** — 여러 병원에서 각기 다른 약을 처방받지만, 기존 DUR(의약품안전사용서비스)은 처방·조제 시점의 B2B 시스템이라 환자가 사전에 확인할 수 없다.
2. **처방약 ↔ OTC/건강기능식품** — 약국·편의점에서 직접 구매하는 일반의약품과 건강기능식품은 DUR이 구조적으로 포착 불가. 가장 위험한 사각지대다. (예: 혈압약 복용 중 이부프로펜 구매 → 위장관 출혈 위험)
3. **OTC ↔ OTC** — 성분 중복을 일반인이 알 방법이 없다. (예: 감기약 속 아세트아미노펜 + 타이레놀 → 과다복용)

추가로, 시니어의 약물 정보 이해도는 42%에 불과하고, 복잡한 약 이름을 앱에 직접 입력하는 것은 사실상 불가능하다.

## Target User — Who is this for?

### 1차 타겟 (MVP): 부모님을 돌보는 성인 자녀

> **페르소나**: 김서연, 38세, IT기업 마케팅 팀장. 서울 거주, 부모님은 경기도 수원(별거).
> 아버지(68세) — 고혈압+당뇨+전립선비대증, 2개 병원, 처방약 7종 + OTC 2~3종.
> 어머니(66세) — 고혈압+골다공증+만성위염, 2개 병원, 처방약 6종 + OTC 2~3종.

**핵심 니즈**: 전문 지식 없이도 부모님의 약물 안전을 원격으로 확인할 수 있는 도구.

**Pain Points**:
- 부모님이 "그냥 혈압약"이라고만 말해 정확한 성분 파악 불가
- 약국 구매약은 처방 기록에 남지 않아 추적 불가
- 약 이름을 알아도 성분 간 상호작용 판단 불가
- 맞벌이·별거로 매 진료 동행 불가

### 2차 타겟 (확장): 다제약물 관리 기관 근무자 (보건소 간호사 등)

MVP 범위 밖. 1차 타겟으로 코어 파이프라인 검증 후 확장.

## 3 Core Features — What does it do?

### Feature 1: 약물 사진 촬영 → 자동 데이터 추출

처방전, 약봉투, OTC 박스, 건강기능식품 라벨 — 어떤 형태든 사진만 찍으면 AI Vision이 약품명·성분명·용량·투약일수를 자동 추출하여 정형 데이터로 저장한다. 사용자가 약물 유형을 구분하거나 이름을 타이핑할 필요 없다.

- **입력**: 카메라 촬영 이미지
- **출력**: `{ name, ingredient, dose, days, type(처방약/OTC/건강기능식품) }`
- **API**: `POST /api/v1/ocr/parse`

### Feature 2: 멀티소스 통합 충돌 분석

처방약 + OTC + 건강기능식품을 하나의 통합 약물 리스트로 병합하고, 신규 약물 등록 시마다 기존 전체 리스트와 자동 대조하여 성분 중복·병용 금기·용량 초과를 검출한다.

- **입력**: 기존 약물 ID 배열 + 신규 약물 ID 배열
- **출력**: 위험도 점수, 충돌 이유, 행동 가이드
- **API**: `POST /api/v1/analysis/check-interaction`
- **결과 분류**: `[안전]` / `[주의]` / `[금기]`

### Feature 3: 쉬운 언어 위험 안내 + 행동 가이드

LLM(Claude)을 활용하여 전문 약학 정보를 비전문가 눈높이의 쉬운 문장으로 변환한다. 단순 경고가 아니라 **"다음 진료 시 의사에게 이렇게 말씀하세요"** 같은 구체적 행동 가이드를 포함한다.

- 안전 시: "충돌 위험이 없습니다. 안심하고 복용하세요."
- 주의/금기 시: 위험 메시지 + 이유 설명 + 구체적 행동 가이드
- 예시: "아버지가 드시는 혈압약(암로디핀)과 새로 처방된 약(탐스로신)은 병용 시 저혈압 위험이 있습니다. 다음 진료 시 의사에게 현재 복용 중인 혈압약을 알려주세요."

## Scope Limitations — What does it NOT do?

1. **의료 행위가 아니다** — 의학적 진단·처방 변경이 아닌 '정보 제공' 서비스. "확인이 필요합니다, 약사에게 문의하세요" 같은 보수적 가이드 원칙. 면책 조항 필수.
2. **손글씨 처방전은 지원하지 않는다** — 비정형 손글씨 인식은 정확도 보장 불가. 인쇄물만 대상.
3. **2차 타겟(관리사) 기능은 MVP에 포함하지 않는다** — 다수 대상자 대시보드, 자문약사 연계 리포트, 방문 간 변동 추적 등은 1차 타겟 검증 후 확장.
4. **복약 알림·스케줄링은 하지 않는다** — 기존 앱들이 이미 제공하는 단순 알람 기능은 범위 밖. 약물 충돌 분석에만 집중.
5. **가족 프로필은 최소 구조만 지원한다** — MVP에서는 "아버지/어머니" 등 간단한 프로필 분리만. 복잡한 권한 관리나 다수 가족 지원은 후순위.
