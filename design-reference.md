# Design Reference: 다제약물 충돌 방지 서비스

> 이 문서는 Claude Coding 및 디자인 작업 시 시각적 방향성을 제공합니다.
> benchmark.md의 UX Patterns와 함께 참고하세요.

---

## 1. Visual Style Direction (시각 스타일 방향)

### 1-1. 디자인 키워드

```
차분한 (Calm)  ·  신뢰감 (Trustworthy)  ·  명료한 (Clear)  ·  따뜻한 (Warm)
```

이 서비스는 **의료 불안을 줄이는 도구**다. 사용자(자녀)는 부모님 약물에 대한 걱정을 안고 앱을 열며, 결과를 빠르게 이해하고 안심하길 원한다. 디자인의 모든 결정은 "이 화면이 불안을 줄여주는가?"를 기준으로 판단한다.

### 1-2. 핵심 스타일 원칙

| 원칙 | 설명 | 금지 사항 |
|------|------|-----------|
| **미니멀리즘** | 한 화면에 하나의 목적. 불필요한 장식 요소 배제. | 그라데이션 남용, 복잡한 일러스트, 장식용 아이콘 |
| **여백 우선** | 요소 간 충분한 여백으로 시각적 숨쉬기 공간 확보. | 빽빽한 정보 배치, 화면 가득 채우기 |
| **카드 기반 UI** | 정보를 카드 단위로 분리하여 시각적 그룹핑. | 긴 스크롤 텍스트, 테이블 뷰 나열 |
| **아이콘 + 텍스트** | 모든 아이콘에 텍스트 라벨 병기. 아이콘 단독 사용 금지. | 아이콘만으로 의미 전달 시도 |
| **둥근 모서리** | 카드·버튼에 부드러운 border-radius (12~16px). | 날카로운 직각 모서리 |

### 1-3. 무드 & 톤

```
"동네 약국의 친절한 약사가 쉬운 말로 설명해주는 느낌"
 — 전문적이지만 딱딱하지 않고, 따뜻하지만 가볍지 않은.
```

- 의료 앱이지만 **병원처럼 차갑지 않다** — 부드러운 컬러, 둥근 형태, 여유로운 여백
- 정보 앱이지만 **데이터 대시보드처럼 복잡하지 않다** — 핵심 결과만 먼저 보여주고, 상세는 탭하면 펼쳐짐
- 가족 돌봄 앱이므로 **차분한 따뜻함**이 기조 — 차가운 블루 단독이 아니라 웜 톤 악센트를 곁들임

## 2. Color Theme (컬러 테마)

### 2-1. 프라이머리 팔레트

```
┌─────────────────────────────────────────────────────────┐
│  Primary Blue      #3B82F6   ████████   메인 액션, 링크    │
│  Primary Blue Dark #2563EB   ████████   버튼 pressed      │
│  Primary Blue Light#DBEAFE   ████████   배경 하이라이트     │
└─────────────────────────────────────────────────────────┘
```

**Primary Blue (#3B82F6)** — 의료 분야에서 가장 보편적으로 사용되는 신뢰·안정의 색상. 차분하면서도 선명하여 CTA 버튼과 주요 인터랙션에 적합.

### 2-2. 시맨틱 팔레트 (트래픽 라이트)

충돌 분석 결과를 3단계로 전달하는 핵심 컬러. 이 컬러들은 **결과 화면에서만** 사용한다.

```
┌─────────────────────────────────────────────────────────┐
│  Safe Green        #10B981   ████████   안전 상태          │
│  Safe Green BG     #D1FAE5   ████████   안전 카드 배경      │
│                                                          │
│  Caution Amber     #F59E0B   ████████   주의 상태          │
│  Caution Amber BG  #FEF3C7   ████████   주의 카드 배경      │
│                                                          │
│  Danger Red        #EF4444   ████████   금기 상태          │
│  Danger Red BG     #FEE2E2   ████████   금기 카드 배경      │
└─────────────────────────────────────────────────────────┘
```

**사용 규칙**:
- Red는 금기(contraindicated) 경고에만 사용. 일반 UI에 빨간색 사용 금지 (불안 유발)
- Amber는 주의(caution) 상태에만 사용
- Green은 안전(safe) 확인에만 사용
- 색상만으로 정보를 전달하지 않음 — 반드시 아이콘 + 텍스트 병기 (색약 대응)

### 2-3. 뉴트럴 팔레트

```
┌─────────────────────────────────────────────────────────┐
│  Gray 900          #111827   ████████   제목, 본문 텍스트   │
│  Gray 600          #4B5563   ████████   보조 텍스트         │
│  Gray 400          #9CA3AF   ████████   비활성, 플레이스홀더 │
│  Gray 200          #E5E7EB   ████████   구분선, 보더        │
│  Gray 50           #F9FAFB   ████████   페이지 배경         │
│  White             #FFFFFF   ████████   카드 배경           │
└─────────────────────────────────────────────────────────┘
```

### 2-4. 보조 컬러

```
┌─────────────────────────────────────────────────────────┐
│  Warm Beige        #FDF6EC   ████████   온보딩·가이드 배경   │
│  Soft Teal         #5EEAD4   ████████   건강기능식품 태그    │
│  Soft Purple       #A78BFA   ████████   OTC 약물 태그       │
│  Soft Blue         #93C5FD   ████████   처방약 태그          │
└─────────────────────────────────────────────────────────┘
```

약물 유형(처방약/OTC/건강기능식품)을 시각적으로 구분하는 태그 색상. 파스텔 톤으로 통일하여 화면 전체의 차분함을 유지.

### 2-5. 컬러 접근성

- 모든 텍스트-배경 조합은 **WCAG AA (4.5:1)** 이상 대비율 준수
- 시맨틱 컬러(안전/주의/금기)는 색상 외에 아이콘(✓ / ⚠ / ✕)으로도 구분
- 다크 모드: MVP에서는 미지원, 추후 확장 고려

## 3. Typography (타이포그래피)

### 3-1. 서체 선택

| 용도 | 서체 | 사유 |
|------|------|------|
| **기본 (한글+영문)** | **Pretendard Variable** | 한국 정부 공공 UI 표준 서체(2024~), iOS/Android 크로스플랫폼 렌더링 일관성, 9가지 굵기 지원, SIL 오픈 라이선스, 한글 자간 조정 불필요 |
| **폴백** | system-ui, -apple-system, sans-serif | 서체 로딩 실패 시 시스템 기본 |

**font-family 설정**:
```css
font-family: "Pretendard Variable", Pretendard, -apple-system, BlinkMacSystemFont, 
             system-ui, Roboto, "Helvetica Neue", "Segoe UI", 
             "Apple SD Gothic Neo", "Noto Sans KR", "Malgun Gothic", sans-serif;
```

### 3-2. 타이포그래피 스케일

시니어 가독성을 고려하여 일반 앱 대비 **1단계 큰 사이즈**를 기본으로 설정.

| 용도 | 사이즈 | 굵기 | line-height | 예시 |
|------|--------|------|-------------|------|
| **대제목 (H1)** | 28px | Bold (700) | 1.3 | "아버지 약물 리스트" |
| **중제목 (H2)** | 22px | SemiBold (600) | 1.35 | "충돌 분석 결과" |
| **소제목 (H3)** | 18px | SemiBold (600) | 1.4 | 카드 내 제목 |
| **본문** | 16px | Regular (400) | 1.6 | 설명 텍스트, 가이드 |
| **본문 강조** | 16px | Medium (500) | 1.6 | 약품명, 성분명 |
| **보조 텍스트** | 14px | Regular (400) | 1.5 | 날짜, 메타 정보 |
| **캡션** | 12px | Regular (400) | 1.4 | 면책 조항, 출처 |

**규칙**:
- 본문 최소 16px — 14px 미만 텍스트는 캡션·면책 용도 외 사용 금지
- 한 화면에 3가지 이내 사이즈만 사용 — 시각적 위계 단순화
- 행간(line-height)은 본문 1.6 이상 — 시니어 가독성 확보

## 5. 컴포넌트 스타일 가이드

| 타입 | 스타일 | 사용처 |

## 7. 아이콘 & 일러스트레이션

### 아이콘 스타일
- **라인 아이콘** (stroke 1.5~2px), 솔리드 아이콘 사용 금지
- 참고 라이브러리: **Lucide Icons** (React Native 호환, 깔끔한 라인 스타일)
- 컬러: 기본 Gray 600 (#4B5563), 활성 상태 Primary Blue (#3B82F6)
- 사이즈: 24px (네비게이션), 20px (카드 내)

### 일러스트레이션 (최소한 사용)
- 빈 상태(empty state)에만 심플 일러스트 사용
  - 예: "등록된 약물이 없습니다" → 약 봉투 라인 일러스트 + 카메라 아이콘
  - 예: "충돌 위험이 없습니다" → 체크 마크 + 방패 라인 일러스트
- 스타일: 단색 라인 드로잉, 브랜드 컬러(Primary Blue) 1색만 사용
- 복잡한 캐릭터·사실적 일러스트 사용 금지

## 8. 간격 & 그리드 시스템

### 간격 단위 (4px 기반)

| 토큰 | 값 | 용도 |
|------|-----|------|
| `xs` | 4px | 아이콘과 텍스트 사이 |
| `sm` | 8px | 태그 내부 패딩, 인접 요소 간 |
| `md` | 12px | 카드 간 간격 |
| `lg` | 16px | 카드 내부 패딩, 섹션 내 여백 |
| `xl` | 24px | 섹션 간 여백 |
| `2xl` | 32px | 주요 블록 간 여백 |
| `3xl` | 48px | 페이지 상·하단 여백 |

### 화면 레이아웃

- 좌우 패딩: 20px (모바일 표준)
- 하단 네비게이션 높이: 60px + safe area
- 카드 최대 너비: 화면폭 - 40px (좌우 패딩)
- 스크롤 방향: 수직 단방향 (수평 스와이프 최소화)
