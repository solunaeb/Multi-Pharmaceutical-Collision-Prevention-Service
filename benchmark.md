# Benchmark: 다제약물 충돌 방지 서비스 레퍼런스

## 1. Similar Services (유사 서비스 분석)

### 1-1. DUR (의약품안전사용서비스) — 한국 건강보험심사평가원

**서비스 개요**: 의사·약사가 처방·조제 시 병용금기, 연령금기, 임부금기 등 의약품 안전성 정보를 실시간으로 점검하는 국가 B2B 시스템. 2008년 시범사업 시작, 현재 전국 의료기관·약국에 적용.

**핵심 기능**:
- 처방·조제 시점에 환자의 투약이력과 DUR 기준을 비교하여 0.5초 이내 경고 메시지 제공
- 병용금기, 효능군중복, 용량주의, 투여기간주의, 노인주의 등 다양한 점검 항목
- '내가 먹는 약! 한눈에' — 최근 1년 처방 투약이력 조회 서비스 (본인인증 필요, 2025년 카카오톡 간편인증으로 7단계→1단계 간소화)

**우리 서비스와의 차별점**:
- DUR은 B2B 시스템으로, 환자가 처방 전 사전 확인 불가
- OTC·건강기능식품은 구조적으로 커버 불가 (가장 큰 사각지대)
- '내가 먹는 약! 한눈에'도 처방·조제 이력만 조회 가능, 상호작용 분석 기능 없음
- 전문 용어 기반 — 일반인/시니어 이해도 낮음

**참고할 점**: DUR의 약물 상호작용 데이터베이스 구조 (병용금기, 효능군중복 등 점검 유형 분류 체계)

### 1-2. Medisafe — 글로벌 복약 관리 앱

**서비스 개요**: 전 세계 1,000만+ 사용자를 보유한 복약 알림·관리 플랫폼. 약물 상호작용 경고, 가족/보호자 연동(Medfriend), 건강 측정값 추적 등을 제공.

**핵심 기능**:
- 복약 알림 및 스케줄링 (복잡한 투약 일정 지원)
- Drug-to-drug interaction checker (4단계: minor → moderate → major → severe)
- Medfriend — 가족/보호자가 복약 누락 시 알림 수신
- 90+ 건강 측정 트래커 (혈압, 혈당, 체중 등)
- 약 모양·색상 시각화로 직관적 식별
- Apple Health 연동, 의료기관 처방 정보 자동 불러오기 (미국 100+ 병원)

**UX 특징**:
- 필박스(pillbox) 스타일 메인 화면 — 일간 복약 현황을 시각적으로 한눈에 파악
- 메인 화면 커스터마이징 가능 (타임라인 / 사각형 분할 / 원형 분할)
- 약의 모양·색상을 시각적으로 선택 — 이름 모르는 약도 시각 단서로 구분 가능
- 깔끔하고 단순한 UI, 차분한 컬러톤

**우리 서비스와의 차별점**:
- Medisafe는 복약 알림이 핵심, 상호작용 체크는 보조 기능
- 약물 입력이 수동 (약 이름 검색·타이핑) — 시니어에게 높은 허들
- 한국 의약품 DB 미지원
- OTC·건강기능식품 통합 분석 기능 부재

**참고할 점**: Medfriend(보호자 연동) 패턴, 위험도 4단계 분류 체계, 필박스 시각화 UX

### 1-3. PillPack (Amazon Pharmacy)

**서비스 개요**: Amazon이 2018년 인수한 온라인 약국 서비스. 복수 처방약을 날짜·시간별로 사전 분류하여 패킷 형태로 월 1회 배송. 2025년 보호자(Caregiver) 기능 및 Medicare 확장.

**핵심 기능**:
- 처방약을 날짜·시간 라벨이 붙은 개별 패킷으로 사전 분류·배송
- 리필 자동 관리 — 의사·보험사와 직접 연동
- Caregiver 기능 (2025년 신규) — 보호자가 자신의 Amazon 계정으로 환자의 약물 관리
- 24/7 약사 접근 가능
- OTC·비타민·흡입기도 주문 가능

**UX 특징**:
- 극도로 단순한 사용자 경험 — 가입 후 약사가 모든 것을 관리
- 물리적 패킷 디자인 — 날짜·시간이 크게 인쇄된 롤 형태
- Amazon 앱 내 통합 — 익숙한 쇼핑 경험과 동일한 UI 패러다임

**우리 서비스와의 차별점**:
- PillPack은 약국 서비스(배송·조제)이며, 약물 상호작용 분석 서비스가 아님
- 미국 전용 서비스
- 사진 기반 약물 인식 기능 없음

**참고할 점**: Caregiver 연동 패턴 (보호자 초대 → SMS 인증 → 자신의 계정에서 관리), 약물 리스트의 시간축 시각화 방식

### 1-4. '내가 먹는 약! 한눈에' — 건강보험심사평가원

**서비스 개요**: 병원·약국에서 조제받은 최근 1년간 의약품 투약내역을 국민이 직접 조회할 수 있는 서비스. 건강e음 앱 및 심평원 홈페이지에서 이용 가능.

**핵심 기능**:
- 최근 1년 투약이력 (약 이름, 효능, 복용법) 조회
- 개인별 알러지·부작용 정보 입력·관리
- 2025년 카카오톡 간편인증 도입으로 본인인증 7단계 → 1단계 간소화
- 만 14세 미만 자녀 투약이력 법정대리인 조회 가능

**우리 서비스와의 차별점**:
- 처방·조제 이력 조회만 가능 — 상호작용 분석 기능 없음
- OTC·건강기능식품 미포함
- 타인(성인 가족) 약물 조회 불가 — 본인 인증 기반
- 전문 용어 그대로 제공 — 쉬운 언어 변환 없음

**참고할 점**: 투약이력 데이터 구조, 카카오톡 연동 간편인증 UX 플로우

## 4. UX Patterns (깔끔하고 간단한 인터페이스 원칙)

우리 서비스의 1차 타겟(30~40대 자녀)과 실질적 수혜자(60~70대 시니어)를 모두 고려한 UX 원칙.

### 4-1. 카메라 퍼스트 (Camera-First)

> 핵심 원칙: 타이핑 제로. 사진 한 장이 입력의 전부.

- 앱 실행 → 홈 화면 중앙에 카메라 버튼 1개가 가장 눈에 띄게 배치
- 약봉투, OTC 박스, 건강기능식품 라벨 — 구분 없이 찍기만 하면 AI가 자동 분류
- 촬영 후 인식 결과를 카드 형태로 보여주고, 사용자는 "맞아요" 한 번만 탭
- **반면교사**: Medisafe는 약 이름을 검색·타이핑해야 함 → 시니어 사용 불가

### 4-2. 트래픽 라이트 시스템 (Traffic Light)

> 핵심 원칙: 결과는 3초 안에 이해 가능해야 한다.

- 충돌 분석 결과를 신호등 컬러로 즉시 전달:
  - 🟢 **안전** (초록) — "충돌 위험 없음. 안심하고 복용하세요."
  - 🟡 **주의** (노랑) — "확인이 필요합니다. 약사에게 문의하세요."
  - 🔴 **금기** (빨강) — "함께 복용하면 안 됩니다. 즉시 의사와 상담하세요."
- 상세 이유는 접힌 상태(collapsed)로 제공 — 탭하면 쉬운 언어 설명 펼쳐짐
- **참고**: Medisafe 4단계(minor/moderate/major/severe) → 우리는 3단계로 단순화

### 4-3. 쉬운 언어 카드 (Plain Language Card)

> 핵심 원칙: 노인 약물 정보 이해도 42%를 60% 이상으로 끌어올린다.

- 모든 약물 정보·경고를 일상 언어로 변환 (LLM 활용)
- 카드 구조:
  - **첫 줄**: 무엇이 문제인지 (예: "혈압약과 진통제가 부딪혀요")
  - **둘째 줄**: 왜 위험한지 (예: "함께 드시면 위장에 출혈이 생길 수 있어요")
  - **셋째 줄**: 어떻게 해야 하는지 (예: "다음 진료 시 의사에게 '이부프로펜도 먹고 있다'고 말씀하세요")
- 전문 용어는 괄호 안에 병기: "진통제(이부프로펜)"
- **반면교사**: DUR 경고는 "NSAIDs 병용 시 GI bleeding risk 증가" — 일반인 이해 불가

### 4-4. 가족 프로필 분리 (Family Profile)

> 핵심 원칙: 한 앱에서 아버지·어머니 약물을 각각 관리.

- 홈 화면 상단에 프로필 전환 탭 (예: 👤 아버지 | 👤 어머니)
- 각 프로필별 독립된 Active_Meds 리스트
- 신규 약물 촬영 시 "누구의 약인가요?" 프로필 선택 1탭
- **참고**: Medisafe Medfriend — 가족 약물 동기화. PillPack Caregiver — SMS 초대 기반 보호자 관리.

### 4-5. 미니멀 네비게이션 (Minimal Navigation)

> 핵심 원칙: 하단 탭 3개 이하. 모든 핵심 작업은 2탭 이내 완료.

- 하단 네비게이션: **홈(약물 리스트)** | **촬영(카메라)** | **기록(히스토리)**
- 홈: 프로필별 현재 복용 약물 리스트 + 충돌 상태 요약
- 촬영: 카메라 → 인식 → 프로필 배정 → 충돌 분석 (원스텝 플로우)
- 기록: 과거 등록·삭제 히스토리, 충돌 분석 리포트 아카이브
- **참고**: 헬스케어 앱 UX 베스트 프랙티스 — 하단 탭 5개 초과 금지, 3탭 내 핵심 작업 완료 권장

### 4-6. 시각적 안정감 (Visual Calm)

> 핵심 원칙: 의료 앱은 불안을 줄여야 한다.

- 컬러 팔레트: 차분한 블루/그린 계열 기본 + 신호등 컬러는 경고에만 사용
- 충분한 여백(white space) — 정보 밀도를 낮게 유지
- 큰 텍스트 사이즈 (최소 16px body, 시니어 가독성 고려)
- 명확한 대비(contrast ratio) — WCAG AA 이상 준수
- 아이콘에 반드시 텍스트 라벨 병기
- **참고**: Medisafe의 차분한 톤, 필박스 시각화의 직관성. 헬스케어 UI에서는 harsh red/cluttered layout 금지.
